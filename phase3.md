# プロジェクト: 焼肉チェーン発注管理システム MVP (Phase 3)
あなたはシニアフロントエンドエンジニアです。フェーズ2で構築した状態管理と非同期処理の基盤の上に、現場での実運用に耐えうる「堅牢なUX」と「ルーティング」を追加してください。

## 1. 新規追加する技術スタック
- ルーティング: `react-router-dom` (URLによる画面の完全分離)
- アイコン: (引き続きLucide Reactを使用)

## 2. フェーズ3の開発要件 (スコープ)

### A. 画面の完全分離とルーティングの導入
`react-router-dom` を導入し、`App.tsx` を書き換えて以下のURL設計にしてください。
- `/` : ログイン画面（擬似的なモック。ボタン一つで「店舗としてログイン」「センターとしてログイン」を選べるだけの簡易的なもの）
- `/store` : 店舗側 発注画面 (`StoreOrderPage`)
- `/center` : センター側 受注管理画面 (`CenterDashboardPage`)
※これに伴い、ナビゲーションバーの役割を「画面切り替え」から「ログアウトしてルートに戻る」等の機能に変更してください。

### B. データの永続化 (Zustand Persist Middleware)
飲食店では「電波が悪い場所での操作」や「誤操作によるリロード」が多発します。
- `useCartStore.ts` に Zustand の `persist` ミドルウェアを適用し、カートの中身が `localStorage` に自動保存されるようにしてください。
- これにより、ブラウザをリロードしてもカートに入れた商品と数量が復元されるようにします。
- ※ `useOrderStore.ts` （センター側の受注データ）については、フェーズ4で実際のDBと繋ぐため、今回は永続化不要（インメモリのままでOK）です。

### C. 検索・絞り込み機能の追加 (`StoreOrderPage`)
膨大な商品マスタから素早く発注できるようにします。
- 店舗側画面の商品リスト上部に「フリーワード検索バー（例: "カルビ"）」を追加してください。
- 既存の「カテゴリタブ（精肉、サイド等）」と掛け合わせてフィルタリング（絞り込み）ができるようにしてください。

## 3. 実装の進め方と注意事項
1. まず `npm install react-router-dom` を実行してください。
2. `App.tsx` に Router を設定し、モックのログイン画面（`LoginPage.tsx`）を新規作成してください。
3. Zustandの `persist` 実装は、公式ドキュメント（v5準拠）のシンプルな記法に従ってください。
4. 完了後、ターミナルに「Phase 3が完了しました。ブラウザをリロードしてもカートのデータが保持されるか確認してください」と出力してください。
