# プロジェクト: 焼肉チェーン発注管理システム MVP (Phase 1)
あなたはシニアフロントエンドエンジニアです。現場の紙の発注書をDX化するWebアプリケーションの「フェーズ1（UIコンポーネントと静的モックの構築）」を実行してください。
※コンテキスト破綻を防ぐため、今回は複雑なグローバル状態管理やAjax処理の実装は行わず、UIの静的なルーティングとコンポーネント分割に集中してください。

## 1. 技術スタック (2026 Best Practices)
- フレームワーク: React + Vite
- 言語: TypeScript (型安全性を確保し、KやPなどの独自単位のバグを防ぐため)
- スタイリング: Tailwind CSS
- UIコンポーネント設計: Atomic Designの概念を緩やかに適用し、再利用可能な部品（Button, Card, Input）を `src/components/ui/` に分離すること。
- アイコン: Lucide React

## 2. ディレクトリ構成の要件
以下の構造でファイルを分割してください。

```text
src/
  ├── components/
  │    ├── ui/ (汎用的なボタンやバッジ等のUI部品)
  │    ├── store/ (店舗側専用のコンポーネント群)
  │    └── center/ (センター側専用のコンポーネント群)
  ├── pages/ (画面全体を管理するView)
  │    ├── StoreOrderPage.tsx
  │    └── CenterDashboardPage.tsx
  ├── types/ (TypeScriptの型定義: Item, Order等)
  └── data/ (静的なモックデータ: mockData.ts)
```

## 3. フェーズ1の開発要件 (スコープ)
### A. 型定義とモックデータの作成 (`src/types/`, `src/data/`)
- 実際の発注書に基づき、`unit` は "K", "P", "1束(100枚)", "本", "個" などの文字列を許容する型を定義してください。
- カテゴリ（精肉、弁当関係、限定メニュー等）を定義し、数十件のダミーデータを作成してください。

### B. 店舗側UI (`StoreOrderPage.tsx`)
- モバイル・タブレットでの操作を前提としたレスポンシブデザイン。
- 左側（または上部）にカテゴリ切り替えタブ。
- メインエリアに商品リスト。各行に「商品名」「規格」「発注数入力欄（＋/ーボタン付き）」を配置。
- ※フェーズ1では、＋/ーを押してもローカルのState（useState）が動くだけで、APIへの送信処理はモック関数（`console.log`）で留めてください。

### C. センター側UI (`CenterDashboardPage.tsx`)
- PC/タブレットでの閲覧を想定。
- 店舗からの発注データ（モック）をリスト表示。
- 注文詳細を開くと、「発注数」と「納品数（入力可能）」が並ぶテーブルUIを表示。
